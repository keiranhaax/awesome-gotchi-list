# Contributing

Thanks for helping improve this list.

Please suggest projects that are genuinely useful to people interested in Tamagotchi, Pwnagotchi, virtual pets, care-loop games, embedded companions, desktop pets, AI pets, or related reverse-engineering and modding work.

## How to Contribute

1. Check the existing list to avoid duplicates.
2. Open a pull request with your suggested addition or cleanup.
3. Follow the entry format exactly and keep the tone objective.

## Criteria

- Add resources that are well documented, historically important, actively maintained, or unusually creative.
- Prefer open-source repositories and durable project pages over short-lived posts.
- Avoid duplicates by checking the existing README before submitting.
- Use one Markdown link per bullet.
- Write one concise, objective sentence that starts with a capital letter and ends with a period.
- Do not include star counts, update dates, hype, or marketing claims unless they are directly relevant and freshly verified.
- For WiFi, security, and capture-related projects, keep descriptions educational and remind users to operate only on networks they own or have permission to test when needed.
- Follow the official [Awesome list guidelines](https://github.com/sindresorhus/awesome/blob/main/pull_request_template.md).

## Entry Format

```markdown
- [Project Name](https://example.com/) - Describes what makes the project useful in one clear sentence.
```

## Examples

Good:

```markdown
- [ExampleGotchi](https://github.com/example/examplegotchi) - Runs a documented ESP32 virtual pet with persistent care stats and an OLED interface.
```

Also good:

```markdown
- [Terminal Pet](https://github.com/example/terminal-pet) - Adds a terminal virtual pet with hunger, mood, autosave, and time-based stat changes.
```

Needs improvement:

```markdown
- [Cool Pet](https://github.com/example/cool-pet) - awesome tamagotchi project!!!
```

That entry needs an objective description, proper capitalization, and a period.

## Quality Bar

This is an awesome list, not a complete directory. Projects should have enough documentation that a reader can understand what they do and why they are notable.
